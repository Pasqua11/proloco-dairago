Sezione 1
Accesso al sistema

Aprire il browser e navigare all'indirizzo dell'applicazione (es. http://localhost:5173 in locale o l'indirizzo di produzione fornito dall'amministratore).

Nella schermata di login inserire nome utente e password. Sono presenti dei pulsanti di accesso rapido per i ruoli preconfigurati, utili per il personale durante l'evento.

ℹ️ Durata sessione: il token di accesso è valido per 8 ore. Allo scadere del tempo il sistema reindirizza automaticamente alla pagina di login.
👥
Sezione 2
Ruoli utente

L'applicazione usa un sistema di permessi basato su ruoli. Le funzionalità visibili si adattano automaticamente al ruolo dell'utente connesso.

🛡️
Admin

Sezione 3
Pannello principale

Dopo il login si accede al pannello di gestione articolato in:

🔝 Barra superiore
Titolo evento, utente connesso, pulsante di logout.
📅 Selettore data/turno
Scelta della data evento e del turno attivo (Pranzo / Cena).
👨‍👩‍👧 Scheda Gruppi
Elenco completo delle prenotazioni e dei gruppi.
🗺️ Scheda Tavoli
Mappa visiva dei tavoli e assegnazione posti.
🍽️ Scheda Menu
Gestione delle voci di menu per il turno.
⚙️ Impostazioni
Configurazione sessione e disposizione tavoli (admin/tavoli).
💾 Lo stato viene salvato automaticamente nel database con un ritardo di 1 secondo dall'ultima modifica. Un indicatore in alto mostra lo stato: Salvando… / Salvato / Errore.
📅
Sezione 4
Gestione date eventi

Aggiungere una data

  1. Fare clic su Aggiungi data nella scheda Impostazioni.
  2. Selezionare la data dal calendario.
  3. Inserire un'etichetta descrittiva (es. Sabato sera).
  4. Confermare per salvare.
ℹ️ Sono supportate fino a 7 date per evento.

Attivare i turni Pranzo / Cena

Per ogni data è possibile abilitare o disabilitare i singoli turni con gli appositi toggle. Se un turno contiene già dei gruppi assegnati, il sistema chiederà conferma prima di disabilitarlo.

Eliminare una data

⚠️ L'eliminazione di una data è irreversibile e rimuove tutti i dati associati: gruppi, assegnazioni e configurazione tavoli di quella data.

Sezione 5
Gestione gruppi e prenotazioni

Aggiungere un gruppo

  1. Andare nella scheda Gruppi e fare clic su Aggiungi gruppo.
  2. Compilare il modulo: Cognome / Nome gruppo e N° persone sono obbligatori.
  3. Inserire facoltativamente: nome referente, telefono, note (es. allergie, esigenze particolari).
  4. Selezionare i piatti dal menu con le relative quantità.
  5. Fare clic su Salva.

Modificare o eliminare un gruppo

Usare l'icona ✏️ matita per modificare o il 🗑️ cestino per eliminare. L'eliminazione richiede conferma; se il gruppo aveva un tavolo assegnato, l'assegnazione viene rimossa automaticamente.

Ricerca e ordinamento

Usare la barra di ricerca per filtrare per nome, telefono o note. Fare clic sulle intestazioni delle colonne per ordinare l'elenco per: nome, persone, telefono, note, tavolo assegnato.

Sezione 6
Assegnazione tavoli

Configurazione tavoli

Prima di assegnare i gruppi, configurare la disposizione fisica dalla scheda Impostazioni:

  • Attivare le file da usare (A→F, fino a 6 file).
  • Per ogni fila impostare il numero di tavoli e i posti per tavolo.
FilaColoreNote
AAmbraPrima fila, solitamente ingresso
BArancione
CRosa/Rosso
DVerde smeraldo
EAzzurro
FViolaUltima fila

Assegnazione manuale

  1. Selezionare il gruppo dall'elenco.
  2. Fare clic sul tavolo desiderato nella mappa visiva.
  3. Specificare i posti da occupare se il gruppo si distribuisce su più tavoli.

Assegnazione automatica

Fare clic su Assegna automaticamente. Il sistema distribuisce i gruppi ottimizzando l'occupazione. È possibile rieseguire per ottenere configurazioni diverse.

📋 Ogni tavolo mostra in tempo reale: posti occupati/totali, nome dei gruppi assegnati e colore della fila.

Copiare una sessione su un'altra data

Dal pannello impostazioni usare la funzione Copia sessione per duplicare gruppi e assegnazioni su un'altra data/turno.

🍽️
Sezione 7
Menu

Accessibile dalla scheda Menu. Le voci sono associate alla sessione corrente (data + turno).

Aggiungere una voce

  1. Fare clic su Aggiungi voce.
  2. Compilare: nome, categoria, prezzo, disponibilità, ordine di visualizzazione.
  3. Salvare.

Gestire le voci

  • ✏️ Modifica — aggiornare nome, prezzo o categoria.
  • Toggle Disponibile — abilitare/disabilitare una voce senza eliminarla.
  • 🗑️ Elimina — rimuovere voci non più necessarie.

Le voci attive vengono mostrate durante l'inserimento/modifica di un gruppo per la selezione dei piatti con quantità.

Sezione 8
Esportazione e stampa

Esportazione PDF

Fare clic sul pulsante Esporta PDF disponibile nella scheda Tavoli o Gruppi. Il file generato include:

  • Il logo della Proloco Dairago
  • Elenco gruppi con dettagli e selezioni dei piatti
  • Mappa assegnazione tavoli per fila
  • Data e turno selezionati

Stampa diretta

Usare il pulsante Stampa per ottenere una versione ottimizzata per la stampa della disposizione dei tavoli.

Sezione 9
Funzioni amministratore

Sblocco PIN

Alcune operazioni sensibili richiedono l'inserimento di un PIN amministratore. Il pannello appare automaticamente quando richiesto.

🔑 Il PIN è configurato dall'amministratore di sistema. Conservarlo in modo sicuro.

Gestione utenti

Accessibile solo con ruolo admin dal menu utenti:

  • Creare nuovi utenti con nome, password e ruolo
  • Modificare password e ruolo di utenti esistenti
  • Disattivare account non più necessari

Sezione 11
Avvio dell'applicazione

Avvio rapido (Windows)

Fare doppio clic sul file avvia.bat nella cartella principale del progetto. Il batch avvia automaticamente backend (porta 3001) e frontend (porta 5173).

Avvio manuale

Aprire due terminali separati ed eseguire:

# Terminale 1 — Backend
cd backend
npm run dev

# Terminale 2 — Frontend
cd frontend
npm run dev

Aprire il browser su http://localhost:5173.

Prerequisiti

  • Node.js installato (versione 18 o superiore)
  • PostgreSQL in esecuzione con il database configurato
  • File .env nel backend con le variabili d'ambiente corrette
VariabileDescrizione
DATABASE_URLStringa di connessione PostgreSQL
JWT_SECRETChiave segreta per la firma dei token
JWT_EXPIRES_INDurata del token (default: 8h)
PORTPorta del backend (default: 3001)
FRONTEND_URLURL del frontend per CORS
💾 Backup: eseguire regolarmente un backup del database PostgreSQL prima di ogni evento.

Produzione

In produzione l'applicazione è servita tramite Nginx come reverse proxy. Fare riferimento alla configurazione nella cartella nginx/ del progetto.
